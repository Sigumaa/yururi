# MEMORY.md

長期的に保持する価値のある情報だけを要約して記録する。

## 運用ルール

- 時刻・日付・曜日などのタイムスタンプ情報は原則書かない
- 毎ターン更新しない。再利用価値が高い新事実だけ追記する
- user_id単位で整理し、関係する項目を更新する
- 軽微な追記はappend、全体整理はreplaceを使う

## 推奨構造

### Users

- user:<id> の好み/地雷/期待値

### Channels

- channel:<id or name> の運用ルール

### Global

- 全体ルール・再発防止メモ

## 含める内容

- ユーザー固有の好みや定義の要約
- チャンネル運用上の重要ルールの要約
- 定期タスクの背景と目的
- 再発防止のための教訓

## 含めない内容

- 会話本文の生ログ
- 単発の挨拶
- 実行済みで再利用価値が低い詳細
